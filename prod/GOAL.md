# GOAL проекта: Кейс «AI-ассистент для подготовки к экзаменам»

## Цель проекта
Спроектировать **MVP** веб-приложения — AI-ассистента для подготовки к ОГЭ и ЕГЭ, который поможет школьникам 8–11 классов:
- находить пробелы в знаниях,
- объяснять ошибки,
- строить персональный путь к результату.

Приложение должно не просто проверять знания, а **учить мыслить** и давать обратную связь, сравнимую с репетиторской, но в доступном цифровом формате. Приложение будет представлять собой адаптивный сайт, оптимизированный для мобильных устройств, с цветовой схемой, вдохновленной GitHub (темная и светлая темы) и желтыми акцентами.

---

## 1. Проблемное заявление

**Целевая аудитория:**
- Школьники 8–11 классов, готовящиеся к ОГЭ и ЕГЭ.
- Учащиеся из регионов с ограниченным доступом к качественному репетиторству.
- Семьи со средним и ниже среднего доходом, для которых услуги репетиторов (1000–3000 руб./час) финансово недоступны.

**Конкретная проблема:**
При подготовке к экзаменам ученики сталкиваются с отсутствием персонализированной обратной связи. Существующие решения (YouTube-лекции, сборники заданий, готовые приложения) предоставляют материалы, но **не анализируют ошибки конкретного ученика** и не строят индивидуальную траекторию обучения. Ученик может решить 100 задач, но не понять, в чём именно его системная ошибка.

**Почему существующие решения не решают проблему полностью:**
1. **YouTube и видеолекции** — пассивное обучение, нет интерактива и проверки понимания.
2. **Банки заданий (РешуОГЭ, ФИПИ)** — есть задачи и ответы, но нет объяснения, почему ответ неверный.
3. **Репетиторы** — эффективно, но дорого (от 1000 руб./час) и недоступно в удалённых регионах.
4. **Готовые приложения** — часто ограничены одним предметом, требуют подписки, не интегрируют AI-аналитику.

**Последствия нерешённой проблемы:**
- Пробелы в знаниях накапливаются, что ведёт к низким баллам на экзаменах.
- Неравенство в доступе к образованию: «золотой миллион» имеет репетиторов, остальные — только самоучки.
- Рост тревожности и выгорания из-за непонимания, как эффективно готовиться.

## 2. Анализ актуальности

**Масштаб проблемы:**
- ОГЭ в 2025 году сдавали **~1,4 млн человек**, ЕГЭ — **~700 тыс. человек** ежегодно.
- Рынок репетиторства в России оценивается в **~200 млрд руб./год**.
- До 40% школьников не могут позволить себе репетиторов (данные опросов ВЦИОМ, 2024).

**Почему именно сейчас:**
1. **Эпоха AI:** Gemini API (и аналоги) делает AI-репетиторство технологически реализуемым с минимальными затратами — стоимость одного объяснения может составлять копейки.
2. **Мобильные устройства:** 95% школьников имеют смартфоны — приложение доступно без покупки ПК.
3. **Serverless-архитектура:** Позволяет запустить MVP с бюджетом <$20/мес (до 512 МБ RAM, БД до 1 ГБ).
4. **Госпрограммы:** Цифровизация образования — приоритет, возможна интеграция с ГИС «Моя школа».

**Риски для ученика при сохранении старого сценария:**
- Снижение средних баллов ОГЭ/ЕГЭ в регионах (разрыв с Москвой — до 20 баллов по ЕГЭ).
- Рост доли платного образования — недоступность вузов для способных, но малообеспеченных учеников.
- Усиление репетиторского «пузыря» — инфляция цен на репетиторов без роста качества.

## 3. Проектирование MVP

### Ключевые функции (≤5)

| Функция | Решаемая проблема | Почему важна на старте |
|---------|-------------------|------------------------|
| **1. AI-диагностика знаний** | Ученик не знает свои слабые места | Определяет точку старта и приоритеты |
| **2. Интерактивный тренажёр с объяснением ошибок** | Нет обратной связи при решении | Заменяет репетитора, даёт разбор каждой ошибки |
| **3. Персональный план подготовки** | Хаотичное изучение тем без системы | Структурирует подготовку на неделю |
| **4. Отслеживание прогресса** | Нет мотивации и понимания роста | Визуализация успехов, gamification |
| **5. Админ-панель** | Нет контроля для учителей/родителей | Мониторинг успеваемости группы |

### User Flow

```
Регистрация → Выбор предмета и класса → Диагностический тест (10 вопросов)
→ Анализ ошибок через Gemini AI → Персональный план → Ежедневные тренировки
→ Дэшборд с прогрессом → Настройки темы/профиля
```

### Технологический стек

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| **Бэкенд** | Python + Flask | Лёгкий, быстрый старт, простота деплоя |
| **База данных** | SQLite (файловая БД) | Не требует сервера, <1 ГБ, встроена в Python |
| **AI** | Gemini 2.0 Flash API | Доступный (бесплатный уровень), мощный, быстрый |
| **Фронтенд** | HTML + CSS + Vanilla JS | Минимальный вес, no build tools, быстрая загрузка |
| **Деплой** | Serverless (Railway/Render/Adaptable) | Бесплатный хостинг, до 512 МБ RAM |
| **Конфигурация** | `.env` | Безопасное хранение API-ключей |

### Главные экраны

1. **Страница входа/регистрации** — минималистичная форма с выбором предмета и класса.
2. **Дэшборд** — статистика попыток, средний балл, последние результаты, кнопка «Начать тренировку».
3. **Тренажёр** — выбор предмета → 10 вопросов с вариантами → мгновенное объяснение после каждого ответа через Gemini → итоговый результат и план.

## 4. Оценка потенциала решения

**Что пользователь получает уже в первой версии:**
- Бесплатный AI-репетитор, работающий 24/7.
- Диагностику пробелов без участия человека.
- Персональный план на неделю с учётом слабых тем.

**Развитие через 3–5 лет:**
1. **1–2 год:** Добавление всех предметов ОГЭ/ЕГЭ, расширение базы вопросов до 5000+.
2. **2–3 год:** Геймификация (рейтинги, достижения), соревнования между классами.
3. **3–4 год:** Видеообъяснения от реальных преподавателей, интеграция с ГИС «Моя школа».
4. **4–5 год:** Адаптивное обучение на основе модели знаний ученика (Bayesian Knowledge Tracing).

**Дополнительные аудитории и партнёры:**
- **Учителя** — мониторинг класса через админ-панель.
- **Репетиторы** — инструмент для автоматической проверки ДЗ.
- **Школы** — лицензирование для использования на уроках.
- **Региональные центры оценки качества образования** — диагностика целых классов.

**Главный технологический риск и его смягчение:**
- **Риск:** Зависимость от Gemini API (доступность, стоимость при масштабировании).
- **Смягчение:** Абстракция AI-слоя через единый интерфейс — при необходимости можно переключиться на YandexGPT, GigaChat, OpenAI.

- Веб-приложение (сайт), адаптированное для мобильных устройств (responsive design).
- Цветовая схема: как в GitHub (темная и светлая темы), акцентный цвет - желтый.
- Функционал: настройки, дэшборд для администраторов, регистрация и вход по email.

- Использование Gemini API.
- Техническая документация: `README.md`.
- Управление конфигурацией: `.env` файл.
